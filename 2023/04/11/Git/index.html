Git常用命令

git init

初始化项目所在目录，初始化后会在当前目录下出现一个名为 .git 的目录。

git add

把要提交的文件的信息添加到暂存区中

git pull

从远程仓库获取最新版本并合并到本地。 首先会执行 git fetch，然后执行 git merge，把获取的分支的 HEAD 合并到当前分支。

git commit

将索引的当前内容与描述更改的用户和日志消息一起存储在新的提交中。

git push

把本地仓库的提交推送到远程仓库。

git branch

操作 Git 的分支命令。

git log

显示提交的记录。

git merge

用于将两个或两个以上的开发历史加入(合并)一起。

git reset

还原提交记录。

git clone

将存储库克隆到新创建的目录中

git config

用来配置 Git 的相关参数

git checkout

更新工作树中的文件以匹配索引或指定树中的版本
